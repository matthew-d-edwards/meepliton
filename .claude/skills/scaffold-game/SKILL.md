You are helping a Meepliton contributor scaffold and begin implementing a new game module.
The contributor may not be a developer. Be patient, clear, and concrete.

## Step 1 — Gather information

Ask the contributor for:
1. **Game ID** — lowercase letters and numbers only, no spaces (e.g. `sushigo`, `catan`)
2. **Game name** — the display name (e.g. `Sushi Go`)
3. **Player count** — minimum and maximum
4. **Brief description** — one or two sentences for the game catalogue

If $ARGUMENTS is provided, use it as the game ID and skip asking for it.

## Step 2 — Check prerequisites

```bash
git status
git branch --show-current
```

If on `main` or uncommitted changes exist, resolve that first.

## Step 3 — Create the branch

```bash
git checkout main && git pull origin main
git checkout -b add-{gameId}-game
```

## Step 4 — Run the scaffold

```bash
./scripts/new-game.ps1 -GameId {gameId} -GameName "{GameName}" -MinPlayers {min} -MaxPlayers {max} -Description "{description}"
```

Show the contributor the files that were created and explain what each one is for.

## Step 5 — Orient the contributor

- **C# file** (`src/games/Meepliton.Games.{Name}/{Name}Module.cs`) — game rules
- **React file** (`apps/frontend/src/games/{gameid}/components/Game.tsx`) — board/UI
- **Types file** (`apps/frontend/src/games/{gameid}/types.ts`) — must mirror C# state types
- Their game is already registered and will appear in the catalogue when run locally

## Step 6 — Offer to implement

Ask: "Would you like me to implement the game now? If so, describe how the game works."

If yes, proceed with implementation using the `game-module` skill knowledge.

## Step 7 — Initial commit

```bash
git add src/games/Meepliton.Games.{Name}/ apps/frontend/src/games/{gameid}/ apps/frontend/src/games/registry.ts
git commit -m "Scaffold {GameName} game module

Generated by new-game.ps1. Ready for implementation."
git push -u origin add-{gameid}-game
```

## Step 8 — Next steps

1. Run `dotnet run --project src/Meepliton.AppHost` to see the game in the lobby
2. Open a PR: `gh pr create --title "Add {GameName} game" --base main`
3. Invoke the `/tester` agent to write tests once implemented
