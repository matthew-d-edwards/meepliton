# Usage in Claude Code: /scaffold-game
# This command walks through creating a new Meepliton game module end-to-end.

You are helping a Meepliton contributor scaffold and begin implementing a new game module.
The contributor may not be a developer. Be patient, clear, and concrete.

## Step 1 — Gather information

Ask the contributor for:
1. **Game ID** — lowercase letters and numbers only, no spaces (e.g. `sushigo`, `catan`, `pandemic`)
2. **Game name** — the display name (e.g. `Sushi Go`, `Catan`, `Pandemic`)
3. **Player count** — minimum and maximum players
4. **Brief description** — one or two sentences for the game catalogue

If you don't have all four, ask before proceeding.

## Step 2 — Check prerequisites

Run these checks before scaffolding:

```bash
# Make sure we're on a clean branch
git status
git branch --show-current
```

If the contributor is on `main` or has uncommitted changes, stop and resolve that first:
- If on main: `git checkout -b add-{gameid}-game`
- If uncommitted changes: ask what they want to do with them

## Step 3 — Create the branch

```bash
git checkout main
git pull origin main
git checkout -b add-{gameId}-game
```

## Step 4 — Run the scaffold

```bash
./scripts/new-game.ps1 -GameId {gameId} -GameName "{GameName}" -MinPlayers {min} -MaxPlayers {max} -Description "{description}"
```

Show the contributor the files that were created and explain what each one is for.

## Step 5 — Orient the contributor

Tell them:
- The **C# file** (`src/games/Meepliton.Games.{Name}/{Name}Module.cs`) is where the game rules live
- The **React file** (`apps/frontend/src/games/{gameid}/components/Game.tsx`) is where the board/UI lives
- The **types file** (`apps/frontend/src/games/{gameid}/types.ts`) must mirror the C# state types
- Their game is already registered and will appear in the catalogue when they run locally

## Step 6 — Offer to implement

Ask: "Would you like me to implement the game now? If so, describe how the game works — the rules, how players take turns, and how someone wins."

If they say yes, attach the GAME-MODULE.md skill context and proceed with implementation.
If they want to do it themselves, point them to `.claude/skills/GAME-MODULE.md`.

## Step 7 — Initial commit

Once the scaffold (and optionally initial implementation) is done:

```bash
git add src/games/Meepliton.Games.{Name}/
git add apps/frontend/src/games/{gameid}/
git add apps/frontend/src/games/registry.ts
git commit -m "Scaffold {GameName} game module

Generated by new-game.ps1. Ready for implementation."
```

## Step 8 — Next steps

Tell the contributor:
1. Run `dotnet run --project src/Meepliton.AppHost` to see their game in the lobby
2. When ready, open a PR with `gh pr create --title "Add {GameName} game" --base main --web`
3. CI will run automatically — use `gh pr checks --watch` to follow along
